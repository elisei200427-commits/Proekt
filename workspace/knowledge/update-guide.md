# Как обновить бота до последней версии

Эта инструкция для тех, кто уже установил бота ранее и хочет обновить его до новой версии.

Обновление занимает 1-2 минуты. Все ваши данные (SOUL.md, MEMORY.md, диалоги, настройки) сохранятся — обновляется только код бота. Если что-то пойдёт не так — скрипт автоматически откатит изменения и восстановит старую версию.

---

## Что нового в версии 2.1.0

**Две новые команды:**

- **/connect** — VS Code через бота за минуту. Пишешь команду → бот возвращает ссылку GitHub-авторизации + код → подтверждаешь → бот присылает ссылку `vscode.dev/tunnel/agent-XXXXXXXX`. Открываешь в браузере или подключаешься из десктопного VS Code. SSH-клиент не нужен.

- **/reauth** — переподключить Claude без обращения в поддержку. Раньше: подписка Claude отвалилась → бот молчит. Теперь: пишешь /reauth → авторизуешься на claude.ai → копируешь URL из браузера → бот сам перезапускается с новым токеном.

Плюс в меню Telegram добавлены **/update** (обновить бота) и **/version** (показать версию).

## Как обновиться — три сценария

Выбери тот что подходит под твою ситуацию.

### Сценарий A — у тебя /update уже видна в меню бота

Это значит ты уже обновлялся раньше и в меню есть команда **🔄 Обновить бота**.

1. Открой бота в Telegram
2. Нажми **/update** (через меню или напиши руками)
3. Бот пришлёт «Проверяю обновления...» → «Доступна версия 2.1.0. Обновляю...» → «Готово»
4. Через 1-2 минуты увидишь сообщение что бот обновлён

После обновления **закрой и снова открой чат с ботом** — Telegram подгрузит новое меню (появятся /connect, /reauth, /version).

### Сценарий B — у тебя в меню только /start /stop /reset /settings /status

Это значит у тебя стоит версия 2.0.0 без новых команд в меню. Команда /update **работает**, но её нужно написать **вручную**.

1. Открой бота в Telegram
2. В строке ввода сообщения напиши **`/update`** (со слэшем впереди) и отправь
3. Бот ответит «Проверяю обновления...» → «Доступна версия 2.1.0. Обновляю...» → через минуту «Готово»

После обновления **закрой и снова открой чат с ботом** — в меню появятся новые команды.

Если бот ответил «Unknown command» или вообще не ответил — у тебя более старая версия. Переходи к Сценарию C.

### Сценарий C — /update не работает (старая версия)

Это значит бот ставили давно, до 16 мая 2026, и команды /update в коде ещё нет. Нужно обновить через сервер.

1. Открой VS Code на компьютере
2. Подключись к своему серверу через **Remote-Tunnels** (Ctrl+Shift+P → «Remote-Tunnels: Connect to Tunnel» → выбери имя своего сервера)
3. Открой терминал в VS Code: Ctrl + ` (обратная кавычка, рядом с цифрой 1)
4. Вставь команду:

```
bash /home/agent/.agent/bot/update-bot.sh
```

5. Нажми Enter. Через 1-2 минуты увидишь `=== Update complete! Version: 2.1.0 ===`

Если у тебя нет настроенного VS Code Tunnel — подключись по SSH (через PuTTY, OpenSSH или другой клиент) и выполни ту же команду.

## Что точно сохранится при обновлении

- SOUL.md, MEMORY.md, GOALS.md, USER.md (твоя личность бота)
- Все диалоги с ботом (sessions.json)
- memory/ дневники по дням и knowledge/ заметки
- .env с токенами и API-ключами
- Настройки бота (/settings → модель, часовой пояс, лимит трат и т.д.)

Обновляется **только код бота** (bot/index.js, библиотеки, скрипты, шаблоны).

## Если что-то пошло не так

**Бот не отвечает после обновления.** Скрипт обновления автоматически откатил изменения. Бот работает на старой версии. Если откат тоже не помог — напиши в поддержку курса.

**«Bot directory not found»** в терминале. Скрипт не нашёл папку бота. Возможно ты подключился не к тому серверу. Проверь подключение в VS Code → Remote Explorer.

**«Syntax error in new index.js»** в логах. Обновление отменено автоматически до того как файлы заменились — бот продолжает работать на старой версии. Попробуй позже — скорее всего временная проблема со скачиванием файла.

**/update пишет «У тебя последняя версия» а версия старая.** Скорее всего ты на ветке отличной от main или VERSION файл повреждён. Используй Сценарий C.

**Хочешь откатиться обратно на v2.0.** В папке бота лежит автоматический бэкап:

```
cd /home/agent/.agent/bot
LATEST_BACKUP=$(ls -dt .backup-* | head -1)
cp -r "$LATEST_BACKUP"/* .
systemctl restart agent-bot
```

## Проверка после обновления

Напиши боту в Telegram команду **/version** — он покажет текущую версию. Должно быть `2.1.0`.

Дополнительно проверь что появились новые команды:
- **/connect** — должна прийти ссылка на GitHub-авторизацию (если ещё не подключал) или текущий статус туннеля
- **/reauth** — должны прийти 2 скриншота-инструкции и кнопка авторизации

## Команды для быстрого копирования

**Через бота (если /update работает):**
```
/update
```

**Через терминал на сервере (если /update не работает):**
```
bash /home/agent/.agent/bot/update-bot.sh
```

**Полная переустановка (последний вариант, если ничего не помогло — данные сохранятся):**
```
curl -sL https://raw.githubusercontent.com/Ntmib/jarvis-architect/main/setup-server.sh | bash
```
